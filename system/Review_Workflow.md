# 茄子学习系统｜教研复盘机制

## 启动时间

从 Day5 开始，每周进行一次“教研复盘”。

## 复盘对象

教研复盘不是复盘学生知识点掌握情况，而是复盘教学系统本身。

重点关注：

1. 哪节课知识密度太低。
2. 哪节课推进过快。
3. 哪种讲法最适合茄子。
4. 哪类题最能暴露问题。
5. 教材、课堂练习、Homework 是否匹配。
6. 是否需要调整课程节奏和难度。

## 记录原则

- 只记录真实课堂反馈。
- 不编造课堂效果。
- 不为了完善系统而补写未发生的观察。
- 没有足够证据时，标注“信息不足”。
- 复盘结论必须能追溯到具体课堂、练习、作业、批改或学生反馈。

## 使用方式

每周教研复盘结果用于：

- 优化后续课程设计。
- 优化教材写法。
- 优化课堂练习和 Homework 匹配度。
- 调整课程节奏。
- 调整课程难度。
- 作为后续 v2.1、v2.2 版本迭代依据。

## 每日课程开发审核流程

从 Day4 起，课程开发固定采用以下流程：

```text
Codex 开发
↓
Review Package
↓
ChatGPT 教研审核
↓
根据审核意见修改
↓
最终版
↓
Word
↓
PDF / 打印版
↓
课堂
↓
批改
↓
更新学习档案
```

### Review Package 要求

每次课程正式生成 Word / PDF 前，先生成审核包。

审核包建议包含：

- `Review_Report.md`
- `Change_Log.md`
- 各学科 `Student_Book.md`
- 必要时附 `Class_Practice.md`、`Homework.md`、`Teacher_Guide.md`、`Answer.md`

### 审核原则

- 未经 ChatGPT 教研审核，不生成最终打印版。
- 审核意见必须先落实到 Markdown 审核稿。
- 审核通过后，再统一生成 Word、PDF 和打印版。
- 课堂完成后，再根据真实批改和反馈更新学习档案。

## Day4 起正式教材标准

ChatGPT 完成 Day4 教研审核后，确认 Day4 可作为《茄子学习系统》第一套正式课程模板。以后 Day5、Day6、Day7 及后续课程按以下标准开发。

### Student Book 固定结构

1. 为什么学习
2. 学习目标
3. 知识讲解
4. 方法总结
5. 高考教练告诉你
6. 最容易错的地方
7. 高考风格题
8. 本节知识地图
9. 本节一句话总结

### Teacher Guide 固定要求

- 课堂板书
- 建议讲课节奏
- 教学重点
- 教学难点
- 建议提问

### 课堂设计原则

- 适合 ChatGPT 连续讲授完整知识块。
- 不为了时长降低知识密度。
- 每门课都要服务“高中名师 + 家庭教师”模式。
- 教材、课堂练习、Homework 必须互相匹配。
- 所有优化必须来自真实课堂反馈或 ChatGPT 教研审核，不编造。
