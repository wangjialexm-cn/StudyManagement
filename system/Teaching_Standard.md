# 茄子学习系统｜Teaching Standard

## 文件用途

本文件是后续教材生成的固定标准。

以后 Codex 生成任何教材前，必须先读取本文件，再生成 Markdown 审核稿。

## 永久教材生成流程

1. 读取 `system/Teaching_Standard.md`
2. 读取 `茄子学习档案/ability_profile.md`
3. 读取 `system/root_cause_library.md`
4. 再参考 `mistakes/*.json` 和 `茄子学习档案/mistakes.md`
5. 生成 Markdown 审核稿
6. 等待 ChatGPT 审核
7. 根据审核意见修改
8. 审核通过
9. 生成全部 PDF
10. 更新 GitHub

未经 ChatGPT 审核通过，不提前生成 PDF。

## 原因驱动教学

从 Day6 起，教材生成不再只根据错题本身设计训练。

优先顺序：

1. 能力模型：学生当前能力状态是什么？
2. Root Cause：错误背后的真正原因是什么？
3. 错题：哪些题暴露了这个原因？

教材应针对能力根因，而不是简单重复原题。

例：

- `be increase` 的 Root Cause 是“不知道 increase 是动词”，不是“不会 be doing”。
- 平均速度表达不规范的 Root Cause 是“概念表达能力不足”，不是“不会公式”。

## 教材比例

以后每节课按以下比例设计：

- Student Book：30%
- Class Practice：30%
- Homework：40%

Student Book 不追求越长越好，应服务课堂讲授，把核心讲清楚。

## Student Book

必须包含：

- 学习目标
- 为什么学习
- 知识讲解
- 方法总结
- 易错点
- 知识地图

控制要求：

- 控制在 4～6 页。
- 是正式教材，不是知识点罗列。
- 语言适合初高中衔接阶段。
- 每个知识块要说明“为什么这样做”，不能只给结论。

## Class Practice

要求：

- 15～20 题。
- 由浅入深。
- 至少 20% 为综合提升题。
- 与 Student Book 同步，但不机械重复。
- 课堂练习服务当堂暴露问题。

## Homework

要求：

- 不重复课堂练习。
- 覆盖全部知识点。
- 增加少量复习题。
- 难度略高于课堂练习。
- 适合独立完成。

## Answer

要求：

- 不能只有答案。
- 必须给完整解析。
- 解析要说明关键步骤和易错点。
- 对表达题、翻译题、证明题必须给评分或判断依据。

## Teacher Guide

必须包含：

- 教学目标
- 每页讲课重点
- 提问设计
- 易错提醒
- 时间建议

Teacher Guide 是给 ChatGPT 上课使用的教学指导，不是学生教材复制版。

## 每章固定要求

至少包含：

1. 一道高考风格题。
2. 一道思维提升题。
3. 一道易错题。

## 学科要求

### 数学

- 突出方法。
- 强调步骤、分类讨论、数形结合和证明意识。
- 每节课至少保留一道高考风格题。

### 英语

- 突出主动输出。
- 必须包含翻译、造句、改错或写作中的至少两类。
- 不只讲语法，要让学生真正写句子。

### 物理

- 尽量加入图示。
- 强调概念表达、过程分析和模型建立。
- 计算题要同时训练物理意义解释。

### 化学

- 尽量加入生活化理解。
- 从生活经验过渡到化学概念。
- 强调符号、单位、公式和微观含义之间的关系。

## 记录原则

- 不编造学生学习结果。
- 计划、教材、作业与实际掌握情况分开记录。
- 学生是否掌握，必须等待真实课堂、批改或复测证据。

## v1.1 更新建议

后续每节 Student Book 最后固定增加两个栏目。

### 一、本节一句话总结

用于帮助学生在课后用一句话抓住本节核心。

示例：

- 数学：判断集合关系，先判断是否相等，再判断是否真子集。
- 英语：现在进行时永远是 be + doing。
- 物理：平均速度永远是总路程除以总时间。
- 化学：mol 是连接微观粒子和宏观质量的桥梁。

### 二、本节能力自评

固定格式：

- □ 我听懂了
- □ 我会做基础题
- □ 我会做综合题
- □ 我可以给别人讲
