# 茄子学习系统 v2.0｜Teaching System

## 课堂流程

每节课固定采用以下结构：

1. 学习目标：明确本课结束后学生应能做到什么。
2. 生活案例导入：用真实或容易想象的情境引出知识。
3. 概念建立：说明为什么需要这个概念，解决什么问题。
4. 知识讲解：分层讲解定义、方法、公式、条件与适用范围。
5. 互动思考：设置短问题，让学生先想、先说、先试。
6. 例题精讲：不只给答案，要讲思路入口、关键转折和检查方法。
7. 易错点总结：总结常见误区、混淆点和检查清单。
8. 课堂练习：少量、有梯度、当堂反馈。
9. 课后训练卷：用于巩固和暴露问题，不追求机械堆量。
10. 教师答案：包含答案、解析、评分点和追问建议。

## 批改流程

批改不只判断对错，应记录：

- 错误题号
- 错误类型
- Root Cause
- 学生原始思路
- 正确思路
- 是否提示后能改正
- 是否需要进入错题库
- 对应能力项与 L 等级变化建议

## 教材生成依据｜原因驱动教学

从 Day6 起，教材生成进入“原因驱动教学”模式。

生成教材时优先读取：

1. `茄子学习档案/ability_profile.md`
2. `system/root_cause_library.md`

再参考：

3. `mistakes/*.json`
4. `茄子学习档案/mistakes.md`

原则：

- 先看能力画像和 Root Cause，再看具体错题。
- 教材不是简单复刻错题，而是针对导致错题的能力根因设计讲解、例题和训练。
- 如果表面错误与 Root Cause 不一致，以 Root Cause 为主。
- 例如：`be increase` 的训练重点不是重复 `be doing`，而是词性识别与词汇主动输出。

## 互动规则

1. 先让学生表达想法，再讲标准方法。
2. 学生卡住时，优先给提示，不直接给完整答案。
3. 对“会做但慢”的题，分析卡顿点。
4. 对“看懂但不会独立写”的题，安排复述、仿写或变式。
5. 对“提示后会”的知识点，不直接判定为掌握。

## 英语教学策略｜Lexical Radar

从 Day6 开始，英语课每天增加 `Today's Lexical Radar`。

每次选择 3～5 个重点词，每个词必须训练：

1. 中文意思；
2. 词性识别；
3. 常见搭配；
4. 动词变化；
5. 自主造句。

Day5 教学复盘后确认：`be doing` 已基础掌握，后续降低 `be doing` 重复训练比例，增加词汇深度训练比例。

英语错因判断规则：

- 不把所有英文错误都直接归为语法错误；
- 先判断学生是否知道关键词的词性和句中身份；
- 再判断是否是动词形式、固定搭配或句子结构问题；
- 若词义认识但不能主动入句，优先归入 `vocabulary_issue / output_instability`。

## Day6 起课堂调整

根据 Day5 System Review，Day6 起各科课堂策略调整如下：

- 数学：在集合综合应用中增加证明意识训练，多追问“为什么？”和“为什么不是？”。
- 英语：降低 `be doing` 重复训练比例，增加 `Today's Lexical Radar`。
- 物理：在计算稳定的基础上增加概念表达训练，例如用一句话解释平均速度。
- 化学：基础掌握较好，进入阿伏伽德罗常数阶段。

## 课堂记录规范

每次课后应记录：

- 日期
- 学科
- 主题
- 使用材料
- 完成情况
- 暴露问题
- 课堂表现
- 需要加入错题库的题目
- 下一步调整

课堂记录保存到 `daily/YYYY-MM-DD/课堂记录.md`，重要结论同步到 `茄子学习档案/learning_record.md`。
