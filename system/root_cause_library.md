# Root Cause Library｜原因驱动教学库

## 文件用途

本文件记录茄子学习中错误背后的真正原因。

系统升级目标：

错题 → Root Cause → 能力模型 → 复习计划 → 教材生成

以后教材生成优先读取：

1. `茄子学习档案/ability_profile.md`
2. `system/root_cause_library.md`

再参考：

3. `mistakes/*.json`
4. `茄子学习档案/mistakes.md`

这样教材真正针对能力，而不是只针对题目表面。

## 使用原则

- Root Cause 必须来自真实课堂、批改或 ChatGPT 教研复盘。
- 不能因为一道题错了，就直接推断学生“不会整个知识点”。
- 先区分表面错误和真正原因。
- 一个 Root Cause 可以对应多道错题。
- 一个错题也可能对应多个 Root Cause，但必须标出主因。

## 当前重点 Root Cause

### RC-ENG-001｜词性识别不足

- 证据：Day5 教学复盘确认，学生能正确写出 `is increasing`、`is decreasing`。
- 表面错误：`The number is increase.`、`The price is decrease.`
- 真正原因：不知道 `increase / decrease` 本身就是动词。
- 不是：不会 `be doing`。
- 教学动作：
  - 每日 `Today's Lexical Radar`；
  - 每个重点词训练中文、词性、搭配、动词变化、自主造句。

### RC-MATH-001｜证明意识不足

- 证据：Day5 System Review 要求 Day6 数学增加证明意识训练，增加“为什么？”、“为什么不是？”。
- 表面问题：集合基础掌握较好，但需要从会判断升级到会说明理由。
- 真正原因：证明意识不足。
- 不是：不会集合。
- 教学动作：
  - 课堂例题要求写出判断依据；
  - 每节数学加入至少 1 道“说明理由/证明”题。

### RC-PHY-001｜概念表达能力不足

- 证据：Day4 物理批改中，平均速度公式会用，但定义表达不规范。
- 表面错误：平均速度解释不完整。
- 真正原因：概念表达能力不足。
- 不是：不会公式。
- 教学动作：
  - 计算题后必须补一句物理意义；
  - 增加“请用一句话解释……”题型。

### RC-CHEM-001｜概念理解不足

- 证据：CO₂ 摩尔质量曾写成 40 g/mol。
- 表面错误：计算失误。
- 真正原因：对化学式下标含义和“先乘后加”的概念理解不够稳定。
- 不是：单纯不会计算。
- 教学动作：
  - 化学计算前先解释符号含义；
  - 从阿伏伽德罗常数、mol、摩尔质量建立微观到宏观的连接。

## 后续维护规则

新增错题时，必须同步判断：

1. 表面错误是什么？
2. Root Cause 是什么？
3. 这个 Root Cause 是否已经存在？
4. 如果已存在，追加证据；如果不存在，再新增 Root Cause。
5. 后续教材优先围绕 Root Cause 设计，而不是围绕原题重复刷题。
